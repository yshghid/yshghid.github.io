    BERT 뉴스 분류 #4 모델링 아이디에이션

    BERT 뉴스 분류 #4 모델링 아이디에이션 #

    #2026-03-01

    Conv1D 챕터들에서는 문제의 성격에 따라 레이어 수(2→3), 필터 수(10→15), Dropout 비율(0.3→0.5), 손실 함수(L2Loss→SigmoidCrossEntropy), 평가 지표(Pearson r→ROC-AUC), 다중 입력 구조(Concatenate), 커스텀 제너레이터까지 직접 설계했다. 당뇨병성 망막증에서는 데이터 전처리(Canny 엣지 크롭), 클래스 가중치, 커스텀 증강 레이어(DRAugment), ResNet의 Bottleneck 잔차 블록, GlobalMaxPooling vs Flatten 선택, L2 정규화 강도까지 모두 의도적인 모델링 판단이었다.

    현재 BERT 프로젝트에서는 이런 수준의 구조적 판단이 거의 없다. 기본 분류 헤드를 그대로 쓰고, 기본 손실 함수를 그대로 쓰고, 토큰 길이 최적화도 없고, 레이어 동결 전략이 없다. 이전 모델링의 기술적 의사결정 패턴(전처리 → 모델 구조 → 학습 전략 → 정규화 → 손실/평가)을 따라가면서, BERT 파인튜닝에서 실무적으로 적용할 수 있는 기법들을 정리한다.

    #

    #1 전처리: max_length 최적화

    이전 DNA 분석에서 입력 길이가 101bp와 21bp로 다른 문제에 맞게 조정되었고, 망막증에서도 512×512로 리사이즈했다.

    현재 max_length=512로 되어 있지만, AG News 기사의 실제 토큰 수 분포를 확인해야 한다. 대부분의 뉴스 제목+본문이 128~256 토큰이면 512는 과도하다. 패딩이 과도하게 많아지면 Self-Attention이 [PAD] 토큰에 불필요한 연산을 하고(attention_mask로 무시하더라도 계산 자체는 발생), 학습 속도가 느려지고, 배치 크기를 키울 수 있는 GPU 메모리를 낭비한다.

    # 토큰 길이 분포 분석
lengths = [len(tokenizer.encode(text)) for text in train_texts]
print(f"평균: {np.mean(lengths):.0f}, 95%ile: {np.percentile(lengths, 95):.0f}, 99%ile: {np.percentile(lengths, 99):.0f}")

    95번째 퍼센타일 길이로 max_length를 설정하면 대부분의 정보를 보존하면서 효율이 크게 좋아진다. 예를 들어 95%ile이 180이면 max_length=192 정도로 설정한다. 이건 당뇨병성 망막증에서 원본 이미지의 실제 망막 크기에 맞춰 크롭한 것과 같은 원리다. 불필요한 여백(패딩)을 제거하는 것이다.

    추가로 dynamic padding(배치마다 그 배치 내 최장 시퀀스에 맞춰 패딩)을 적용하면 더 효율적이다. Hugging Face의 DataCollatorWithPadding이 이걸 해준다.

    #

    #2 레이어 동결 전략 (Layer Freezing)

    이전 망막증 모델에서 Dropout 비율을 조절하고 L2 정규화 강도를 선택한 것이 과적합 방지를 위한 구조적 판단이었다.

    BERT의 12개 Transformer 레이어는 역할이 다르다. 하위 레이어(03)는 문법, 형태소 같은 범용적 언어 특징을 인코딩하고, 상위 레이어(811)는 과제에 특화된 의미적 특징을 인코딩한다. 하위 레이어의 범용적 지식은 뉴스 분류에서도 그대로 유용하므로 굳이 바꿀 필요가 없다. 이 레이어들을 동결(freeze)하면 학습 파라미터 수가 줄어서 과적합 위험이 감소하고 학습 속도도 빨라진다.

    # 임베딩 레이어 + 하위 6개 레이어 동결
for param in model.bert.embeddings.parameters():
    param.requires_grad = False
for i in range(6):  # layer 0~5 동결
    for param in model.bert.encoder.layer[i].parameters():
        param.requires_grad = False
# layer 6~11 + classifier만 학습

    이건 이전 챕터에서 Conv1D의 레이어 수를 문제 복잡도에 맞게 23개로 조절한 것과 유사한 판단이다. “이 문제를 풀기 위해 모델의 어느 부분까지 건드려야 하는가?“를 결정하는 것이다. AG News처럼 데이터가 충분하고(12만 건) 과제가 비교적 단순한 경우에는 상위 46개 레이어만 풀어도 충분할 수 있다.

    #

    #3 레이어별 차등 학습률 (Discriminative Learning Rate / Layer-wise LR Decay)

    VAE에서 ExponentialDecay로 시간에 따라 학습률을 줄인 것이 학습 단계별 조절이었다면, 여기서는 레이어 깊이에 따라 조절한다.

    단순 동결은 이진적이다. 학습하거나 안 하거나. 차등 학습률은 더 세밀하다. 하위 레이어에는 아주 작은 학습률(예: 1e-6)을 주고, 상위 레이어에는 더 큰 학습률(예: 3.5e-5)을 주고, 분류 헤드에는 가장 큰 학습률(예: 3.6e-5)을 준다. 하위 레이어의 범용 지식은 살짝만 조정하고, 상위 레이어는 뉴스 분류에 맞게 더 크게 조정하는 것이다.

    Sun et al.(2020)의 논문이 이 기법을 텍스트 분류에 적용해서 효과를 입증했고, 구체적으로는 상위 레이어의 학습률을 설정한 뒤 하위 레이어로 갈수록 0.9 같은 감쇠율을 곱하는 방식을 사용한다.

    # 레이어별 차등 학습률 구현
optimizer_grouped_parameters = []
lr = 3.5e-5
lr_decay = 0.9

# 분류 헤드: 가장 큰 학습률
optimizer_grouped_parameters.append({
    "params": model.classifier.parameters(),
    "lr": 3.6e-5
})

# Transformer 레이어 11→0: 점점 작은 학습률
for i in range(11, -1, -1):
    optimizer_grouped_parameters.append({
        "params": model.bert.encoder.layer[i].parameters(),
        "lr": lr * (lr_decay ** (11 - i))
    })
    # layer 11: 3.5e-5, layer 10: 3.15e-5, ..., layer 0: ~1.0e-5

# 임베딩: 가장 작은 학습률
optimizer_grouped_parameters.append({
    "params": model.bert.embeddings.parameters(),
    "lr": lr * (lr_decay ** 12)
})

optimizer = AdamW(optimizer_grouped_parameters)

    이 내용은 DNA 분석에서 Conv1D 레이어를 쌓을 때 “아래쪽 레이어는 기본 패턴, 위쪽 레이어는 복잡한 패턴"이라는 역할 분담을 인식한 것의 자연스러운 확장이다. 레이어마다 역할이 다르니 학습 강도도 달라야 한다는 판단이다.

    #

    #4 분류 헤드 구조 확장 이전 망막증 모델에서 GlobalMaxPooling → Dense(1024, relu, L2) → Dense(5) 구조를 설계했다. Conv1D 모델에서도 Flatten 후 Dense(1)로 바로 가지 않고 구조를 결정했다.

    기본 BERT 분류 헤드는 [CLS] → Dropout(0.1) → Dense(768→4)로 아주 단순하다. 이 단일 Dense 레이어에 768차원의 풍부한 표현을 4개 클래스로 압축하는 모든 부담이 실린다.

    class EnhancedClassifier(nn.Module):
    def __init__(self, hidden_size=768, num_labels=4, dropout_rate=0.3):
        super().__init__()
        self.dropout1 = nn.Dropout(dropout_rate)
        self.dense1 = nn.Linear(hidden_size, 256)
        self.activation = nn.GELU()
        self.dropout2 = nn.Dropout(dropout_rate)
        self.dense2 = nn.Linear(256, num_labels)
    
    def forward(self, cls_output):
        x = self.dropout1(cls_output)
        x = self.dense1(x)
        x = self.activation(x)
        x = self.dropout2(x)
        return self.dense2(x)

    768 → 256 → 4로 중간 레이어를 추가하면 비선형 변환 능력이 높아진다. Dropout을 0.1에서 0.3으로 올리는 것은, 이전 챕터에서 서열이 짧을 때(21bp) 0.3, 길 때(101bp) 0.5를 쓴 것처럼 과적합 위험도에 맞춰 조절하는 판단이다. 활성화 함수를 ReLU 대신 GELU로 쓰는 건 BERT 내부가 GELU를 쓰므로 일관성을 맞추는 것이다.

    다른 접근으로는 [CLS] 토큰만 쓰지 않고 여러 레이어의 출력을 결합하는 방법이 있다. 마지막 4개 레이어의 [CLS] 출력을 concatenate해서 768×4 = 3072차원을 만들고 여기에 분류 헤드를 붙이면, 각 레이어가 포착한 서로 다른 수준의 특징을 모두 활용할 수 있다. 이건 망막증 모델에서 크로마틴 접근성을 Concatenate로 합류시킨 것과 같은 다중 정보 결합 전략이다.

    #

    #5 정규화 기법: Label Smoothing

    이전 DNA 분석에서 Dropout으로 과적합을 방지하고, 망막증에서 L2 정규화와 Dropout을 동시에 썼다.

    일반적인 cross-entropy에서 정답 레이블은 [0, 0, 1, 0]처럼 하나의 클래스에 확률 1.0을 집중시킨다. 이건 모델에게 “100% 확신을 가져라"라고 요구하는 것인데, 과도한 확신은 과적합으로 이어질 수 있다. Label smoothing은 정답을 [0.025, 0.025, 0.925, 0.025]처럼 약간 부드럽게 만들어서 모델이 지나치게 확신하지 않도록 유도한다.

    training_args = TrainingArguments(
    label_smoothing_factor=0.1,  # 추가
    # ... 기존 설정들
)

    ε=0.1이면 정답 클래스의 목표가 1.0에서 0.9 + 0.1/4 = 0.925로 내려가고, 나머지 클래스는 0에서 0.1/4 = 0.025로 올라간다. 모델이 0.925 이상의 확률을 내놓아도 추가 보상이 없으므로 불필요하게 극단적인 확률을 만들지 않는다.

    #

    #6 Warmup + 학습률 스케줄러 설계

    이전 VAE에서 ExponentialDecay를 써서 에포크마다 0.95배로 줄였다.

    현재 코드에 warmup_ratio가 Optuna 탐색 범위에 있지만, 스케줄러 자체의 설계에 대한 판단은 부족하다. 실무에서는 warmup 후 어떤 스케줄을 쓸지가 중요하다.

    training_args = TrainingArguments(
    warmup_ratio=0.1,             # 전체의 10%는 워밍업
    lr_scheduler_type="cosine",   # 코사인 감쇠 (linear 대신)
    # ...
)

    cosine 스케줄러는 warmup 후 학습률이 코사인 곡선을 따라 부드럽게 감소한다. linear보다 후반부에서 학습률이 더 천천히 줄어들어서 미세한 조정이 더 오래 지속된다. 이전 챕터에서 200 에포크 동안 매 10 에포크마다 성능을 출력하며 학습 곡선을 관찰한 것처럼, 스케줄러의 모양이 학습 역학에 직접 영향을 미친다.

    #

    #7 Gradual Unfreezing (점진적 해동)

    전이 학습에 특화된 고급 기법으로, 핵심 아이디어는 처음에는 분류 헤드만 학습하고, 이게 어느 정도 안정화되면 상위 레이어를 풀고, 점차 하위 레이어까지 풀어나간다. 한 번에 모든 레이어를 학습하면 랜덤 초기화된 분류 헤드의 엉뚱한 그래디언트가 사전학습 가중치를 망칠 수 있다. 단계적으로 풀면 이 위험을 줄인다.

    # 1단계: 분류 헤드만 학습 (1 에포크)
for param in model.bert.parameters():
    param.requires_grad = False
train(epochs=1)

# 2단계: 상위 4개 레이어 해동 (2 에포크)
for i in range(8, 12):
    for param in model.bert.encoder.layer[i].parameters():
        param.requires_grad = True
train(epochs=2, lr=2e-5)

# 3단계: 전체 해동 (1 에포크)
for param in model.bert.parameters():
    param.requires_grad = True
train(epochs=1, lr=5e-6)

    #

    #8 Early Stopping

    이전 DNA 분석에서 매 10 에포크마다 훈련/검증 성능을 출력하며 과적합을 감시했다.

    from transformers import EarlyStoppingCallback

training_args = TrainingArguments(
    evaluation_strategy="steps",
    eval_steps=500,
    load_best_model_at_end=True,
    metric_for_best_model="f1",
    greater_is_better=True,
)

trainer = Trainer(
    callbacks=[EarlyStoppingCallback(early_stopping_patience=3)],
    # ...
)

    검증 F1이 3번 연속 개선되지 않으면 학습을 중단하고 가장 좋았던 체크포인트로 되돌린다. 이건 이전 챕터에서 수동으로 하던 “훈련 vs 검증 성능 비교"를 자동화한 것이다.

    #

    #9 정리

    현재 프로젝트는 “사전학습 모델을 가져와서 기본 설정으로 파인튜닝하고 Optuna로 숫자만 돌린다"에 가깝다. 이전 챕터들에서 보여줬던 “문제와 데이터의 특성을 분석하고, 그에 맞게 모델 구조와 학습 전략을 의도적으로 설계한다"는 DL 모델링의 깊이가 빠져 있다. 위에서 정리한 기법들을 적용하면, Optuna가 탐색하는 공간 자체의 품질이 올라가고, 같은 탐색 횟수로도 더 좋은 결과를 얻을 수 있다.
