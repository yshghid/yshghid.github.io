                      Genome Curation #6 NGS Report Curation - Guide

                      Genome Curation #6 NGS Report Curation - Guide #

                      #2026-04-02

                      다양한 기관에서 수집된 NGS 리포트를 열어, 정해진 양식에 맞게 변이 정보를 추출하고 정리한다. 이때 리포트마다 형식이 다르고, 기록 방식이 다르고, 심지어 같은 항목의 명칭도 다르다. 이 혼란 속에서 통일된 데이터베이스를 구축하려면 명확한 규칙이 필요하다.

                      작업의 전체 구조는 다음과 같다.

                      NGS 리포트 (원본, 기관마다 형식 다름)
        ↓
[1] 검체 및 시퀀싱 정보 추출
[2] SNV 추출 (Tier 1, 2만)
[3] CNV 추출
[4] Fusion 추출
        ↓
통일된 형식의 정제 데이터셋 (CGDB)

                      #

                      #1 검체 및 시퀀싱 정보 추출

                      리포트에서 가장 먼저 추출하는 것은 “이 검체가 무엇이고, 어떻게 분석되었는가"에 대한 메타데이터다.

                      #

                      1-1. 검체 번호 (Specimen ID)

                      연구검체번호를 그대로 기록한다. 예를 들어 S004-004 같은 형식이다. 이 번호가 이후 SNV, CNV, Fusion 데이터 모두를 연결하는 열쇠(key)가 된다. 하나의 검체번호에 여러 변이가 매달리는 구조이므로, 검체번호가 틀리면 모든 데이터의 연결이 깨진다.

                      #

                      1-2. 검체의 질 (Quality)

                      QC pass 여부를 pass, fail, not available 중 하나로 기록한다. pass 판정 기준은 패널마다 다르지만, 일반적인 기준은 다음과 같다.

                      pass 조건 (모두 충족 시):
  - target based coverage > 90%
  - coverage within target region > 90%
  - uniformity > 80%

                      target based coverage란 패널이 타겟으로 삼은 영역 중 실제로 시퀀싱 읽기가 도달한 영역의 비율이다. 90% 이상이어야 한다는 것은, 타겟 영역의 최소 90%에서 의미 있는 수의 reads가 존재해야 한다는 뜻이다. coverage within target region은 타겟 영역 내에서 일정 깊이 이상으로 읽힌 비율이다. uniformity는 타겟 영역 전체에 걸쳐 읽기 깊이가 얼마나 균일한지를 나타낸다. 80% 미만이면 특정 영역은 매우 깊게 읽혔지만 다른 영역은 거의 안 읽힌, 불균일한 상태라는 뜻이다.

                      종양 순도(tumor purity, %) 도 함께 기록한다. 종양 순도는 시료 속에서 종양 세포가 차지하는 비율이다. 순도가 30%라면 DNA의 70%는 정상 세포에서 온 것이므로, 체세포 돌연변이의 VAF가 실제보다 낮게 측정된다. 이 값이 있어야 나중에 VAF를 해석할 때 보정이 가능하다.

                      #

                      1-3. 검사결과: 구조화된 데이터 (TMB, MSI)

                      TMB(종양변이부담) 를 기록한다. 리포트에 SNV, Indel, Large(구조변이) 각각의 TMB가 나뉘어 있는 경우 각각 기록하되, total TMB는 필수다. 리포트에 alpha, beta, gamma로 구분된 TMB가 있는 경우(일부 패널에서 분석 방법에 따라 여러 값을 제시) beta 값을 기록한다.

                      MSI(반복서열불안정성) 를 기록한다. 분류는 세 가지다.

                      • MSS: 불안정성을 보이는 마커가 없음 (Microsatellite Stable)
                      • MSI-L: 1개의 마커에서 불안정성 (Microsatellite Instability-Low)
                      • MSI-H: 2개 이상의 마커에서 불안정성 (Microsatellite Instability-High)

                      리포트에 MSS/MSI-L처럼 코드 체계에 없는 복합 표기가 나오면, 데이터베이스에서 other를 선택하고 텍스트로 직접 입력한다.

                      #

                      1-4. 패널명 & 버전

                      리포트에서 사용된 유전자 패널의 이름과 버전을 기록한다. 이때 반드시 기존 데이터베이스에 등록된 패널인지 먼저 확인한다.

                      패널 기록 의사결정 흐름:

리포트에서 패널명 확인
        ↓
패널수집현황 DB에서 검색
        ↓
  ┌─── DB에 있음 ──→ panel code + 이름 입력 (예: alphaliquid100(v2))
  │
  ├─── DB에 없지만 패널명 알 수 있음 ──→ 이름 그대로 복사 붙여넣기
  │                                        (예: first-cancer panel v3.1)
  │
  ├─── 패널명 모르겠음 ──→ 임시패널정리 시트에 임시명 부여 + 확인 필요 항목 작성
  │                         → 담당자가 확인
  │
  └─── 완전히 새로운 패널 (어떤 기관에서도 사용 이력 없음)
                           → 신규패널 시트에 작성

                      #

                      1-5. 샘플 타입

                      tissue 또는 liquid 중 하나로 기록한다.

                      • tissue로 판정하는 경우: 검체 type이 tumor, 또는 FFPE(Formalin-Fixed Paraffin-Embedded) 인 경우다. FFPE는 조직을 포르말린으로 고정하고 파라핀 왁스에 담가 보존한 것으로, 병리 검사에서 가장 흔한 조직 보존 방법이다. FFPE 처리된 조직에서 DNA를 추출하면 단편화와 화학적 변형이 발생하여 시퀀싱 품질에 영향을 줄 수 있는데, 이것이 artifact 변이의 원인이 되기도 한다.
                      • liquid로 판정하는 경우: 검체 type이 blood이거나, 패널명에 “liquid” 또는 “liquid biopsy"가 포함된 경우다.

                      #

                      1-6. NGS Technique

                      리포트에 명시된 시퀀싱 기법을 확인하여 기록한다.

                      판정 기준:

특정 유전자를 검사한 경우 ──────────────→ targeted sequencing (PCR-amplicon)
리포트에 "amplicon based assay" 명시 ───→ PCR-amplicon
리포트에 "hybrid capture" 명시 ─────────→ hybrid capture enrichment
패널명이 WGS이거나 genome insight ──────→ WES/WGS
RNA 기반 분석 ──────────────────────────→ RNA sequencing

                      #

                      #2 SNV 추출

                      SNV(단일 염기 변이)와 짧은 삽입/결실(Indel)을 추출하는 단계다.

                      #

                      2-1. 검체번호: 연구검체번호를 기록한다. 이 변이가 어떤 검체에서 나왔는지를 연결한다.

                      2-2. 유전자: 변이가 위치한 유전자 이름이다. 예를 들어 EGFR, BRAF, TP53 등이다.

                      2-3. 염기서열 변화 (DNA-Change): HGVS 표기법에 따른 코딩 DNA 수준의 변화다. c. 접두사로 시작한다.

                      예시:
  c.1457_1471del       ← 1457~1471번째 염기가 결실
  c.1302T>A            ← 1302번째 염기가 T에서 A로 치환

                      #

                      2-4. 아미노산 서열 변화 (Amino Acid Change): 단백질 수준의 변화를 두 가지 형식으로 기록한다.

                      3-letter 표기는 아미노산의 세 글자 약어를 사용한다.

                      예시:
  p.Gln486_Thr490del   ← 486번째 글루타민(Gln)~490번째 트레오닌(Thr) 결실
  p.Arg394Ter          ← 394번째 아르기닌(Arg)이 종결코돈(Ter)으로 변환

                      1-letter 표기는 아미노산의 한 글자 약어를 사용한다.

                      예시:
  p.N486_P490del       ← 486번째 N(아스파라긴)~490번째 P(프롤린) 결실
  p.L16Pfs*9           ← 16번째 L(류신)이 P(프롤린)로 바뀌고 프레임시프트,
                          9번째 코돈에서 정지
  p.Ye434*             ← 434번째 Y(타이로신)에서 정지코돈 발생
  p.Q137_P142del       ← 137번째 Q(글루타민)~142번째 P(프롤린) 결실
  p.Q137_P142ins       ← 137번째~142번째 사이에 삽입

                      #

                      2-5. Read depth: 해당 위치를 덮는 총 읽기 수다.

                      2-6. VAF (%): 변이 대립유전자 빈도를 백분율로 기록한다.

                      2-7. 변이 유형 (Variant Type): 8가지 범주 중 하나를 선택한다.

                      코드 체계:
  1: nonsense     ← 정지 코돈 생성
  2: missense     ← 아미노산 변경
  3: frameshift   ← 읽기 틀 이동
  4: inframe      ← 읽기 틀 유지 삽입/결실
  5: splicing     ← 스플라이싱 접합부 변이
  6: others       ← 위에 해당하지 않음
  7: unknown      ← 분류 불가
  8: silent       ← 아미노산 변경 없음 (동의 변이)

                      #

                      2-8. 변이 분류 (Variant Classification / Tier): 임상적 분류를 기록한다.

                      코드 체계:
  1: pathogenic (Tier 1)
  2: likely pathogenic (Tier 2)
  3: uncertain significance (Tier 3)
  4: benign (Tier 4)
  5: likely benign (Tier 5)
  6: others
  7: unknown

                      SNV는 Tier 1, 2에 해당하는 변이만 작성한다.

                      #

                      #3 CNV 추출

                      복제 수 변이(Copy Number Variation)를 추출하는 단계다. SNV와 달리 Tier 제한 없이 기록한다.

                      #

                      3-1. 검체번호: 연구검체번호.

                      3-2. 유전자: CNV의 영향을 받는 유전자 이름.

                      3-3. 복제수 변이 부위 (Locus): 세포유전학적 위치 표기다. 예를 들어 11q13.3은 11번 염색체 장완(q) 1구역 3밴드 소밴드 3이라는 뜻이다. 이것은 현미경으로 염색체 밴딩 패턴을 볼 때 사용하는 전통적 표기법이다.

                      3-4. Chromosome, Start, End position: 정확한 게놈 좌표다. 예를 들어 chr3:178916680은 3번 염색체의 178,916,680번째 위치다.

                      3-5. Copy number alteration type: 복제 수 변화의 유형을 기록한다.

                      선택지:
  gain          ← 복제 수 증가 (일반적으로 CN 3~4)
  loss          ← 복제 수 감소 (CN 0~1)
  amplification ← 고도 증폭 (CN ≥ 5, 보통 gain보다 강한 의미)
  deletion      ← 결실 (loss와 유사하지만 완전 소실 포함)
  other         ← 위에 해당하지 않음

                      gain과 amplification의 차이를 짚어야 한다. gain은 복제 수가 정상(2)보다 약간 늘어난 것(3~4 copy)이고, amplification은 현저하게 늘어난 것(5 copy 이상, 때로는 수십 copy)이다. 임상적으로 amplification이 더 중요한데, 예를 들어 ERBB2(HER2) amplification은 유방암에서 trastuzumab 치료의 적응증이지만, 단순 gain은 그렇지 않을 수 있다.

                      #

                      3-6. Fold change: 정상 대비 읽기 깊이 비율.

                      3-7. Estimated copy number (CN): 추정 복제 수.

                      3-8. Tier: 1, 2, 3, 4 중 하나. 리포트에 로마 숫자(I, II, IIC, IID)로 표기된 경우 이를 아라비아 숫자로 변환한다.

                      3-9. Evidence level: A, B, C, D 중 하나. Tier와 evidence level은 별개의 축이다. Tier는 “이 변이가 임상적으로 얼마나 중요한가"이고, evidence level은 “그 중요성을 뒷받침하는 근거의 강도가 얼마인가"다. 예를 들어 같은 Tier 2라도 evidence level A(전향적 임상시험 결과)와 evidence level D(사례 보고)는 근거의 무게가 다르다.

                      #

                      #4 Fusion 추출

                      유전자 융합을 추출하는 단계다.

                      #

                      4-1. 검체번호: 연구검체번호.

                      4-2. Fusion gene: 융합 유전자 이름. 예를 들어 EML4-ALK처럼 5’ 파트너와 3’ 파트너를 하이픈으로 연결한다.

                      4-3. Gene1: 첫 번째 파트너 유전자. 예를 들어 EML4.

                      4-4. Gene1 breakpoint: 첫 번째 유전자의 절단 좌표. 예를 들어 chr2:42522656.

                      4-5. Gene2: 두 번째 파트너 유전자. 예를 들어 ALK.

                      4-6. Gene2 breakpoint: 두 번째 유전자의 절단 좌표.

                      4-7. Fusion supporting reads 수: 융합 접합부를 가로지르는 읽기의 수. 이 값이 클수록 fusion call의 신뢰도가 높다.

                      4-8. 검출방법: DNA 또는 RNA 중 하나. DNA 기반이면 구조 변이(breakpoint) 수준에서 탐지한 것이고, RNA 기반이면 실제 전사된 융합 mRNA를 직접 읽은 것이다.

                      4-9. Tier: 임상적 중요도 분류.

                      #

                      #5 실무 작업 시 판단이 필요한 주요 케이스

                      케이스 1: 리포트에 Tier가 표기되어 있지 않은 경우

                      리포트에 변이는 있지만 Tier 분류가 없는 경우가 약 절반이다. 이때 변이의 이름 자체로 분류가 가능한 경우(예: BRAF V600E는 명백한 Tier 1)에는 이름 기반으로 분류를 부여한다. 하지만 이것은 “변이를 기능적으로 분석해서 판정한 것"이 아니라 “이름으로 추정한 것"이므로, 이 구분을 인식하고 있어야 한다.

                      케이스 2: 리포트에 모든 변이가 섞여있는 경우

                      리포트에 “Clinically Significant"와 “Unknown Significance” 구분 없이 모든 변이가 한 테이블에 나열된 경우가 있다. 이때는 각 변이를 개별적으로 검토하여 Tier를 판정해야 한다. 리포트의 테이블 이름, 섹션 제목, 변이 옆에 붙은 annotation(예: “therapeutic”, “amplification” 등)을 단서로 사용한다.

                      케이스 3: “Clinically Significant Therapeutic Amplification” 같은 표현

                      리포트에 Tier 번호 대신 서술적 표현으로 분류된 경우가 있다. “clinically significant"는 Tier 1 또는 2에 해당할 가능성이 높고, “therapeutic"은 치료 관련 변이이므로 역시 Tier 1~2에 해당할 가능성이 높지만, 자동으로 Tier 1이나 2로 단정할 수 없다. 이런 경우 리포트 전체 맥락, 해당 패널의 분류 기준, 변이의 구체적 내용을 종합적으로 판단해야 한다.

                      케이스 4: MSI 값이 코드 체계에 없는 형태인 경우

                      MSS/MSI-L처럼 두 값이 슬래시로 연결된 표기가 나오면 데이터베이스의 선택지(MSS, MSI-L, MSI-H)에 해당하지 않으므로 other를 선택하고 원문을 그대로 텍스트로 입력한다.

                      케이스 5: TMB가 여러 값으로 나뉘어 있는 경우

                      일부 패널에서는 TMB를 alpha, beta, gamma로 나누어 보고한다. 이것은 분석 방법(어떤 변이까지 포함하느냐, 어떤 영역을 기준으로 하느냐)에 따른 차이다. 이 경우 beta 값을 기록한다. SNV, Indel, Large 각각의 TMB가 있으면 각각 기록하되, total TMB는 반드시 포함한다.

                      케이스 6: 패널명을 모르는 경우

                      리포트에 패널명이 명시되지 않았거나, 명시된 이름이 기존 DB에 없는 경우 의사결정 흐름을 따른다. 임시패널명을 부여하고 확인이 필요한 항목(분석 유전자 수, 기법 등)을 함께 기록하여 담당자에게 넘긴다. 완전히 새로운 패널인 경우(다른 기관에서도 사용한 적 없는 패널) 신규패널 시트에 별도 기록한다.

                      #

                      #6 전체 작업 흐름 요약

                      [준비]
 리포트 원본 확보
 패널수집현황 DB 확인

                      #7 핵심 요약

                      • 검체 및 시퀀싱 정보는 모든 변이 데이터의 맥락을 제공한다. QC 기준(target coverage >90%, uniformity >80%), TMB(beta 값 기록), MSI(코드에 없으면 other), 패널명(DB 대조 필수), 샘플타입(FFPE→tissue, blood→liquid), NGS기법을 정확히 기록한다.
                      • SNV 추출의 최중요 규칙은 Tier 1, 2만 기록하는 것이다. DNA-Change(c. 표기)와 AA-Change(3-letter, 1-letter 모두)를 빠짐없이 기록하고, 변이유형(8가지)과 변이분류(Tier)를 코드화한다.
                      • CNV 추출은 Tier 제한 없이 모든 복제 수 변이를 기록한다. gain과 amplification의 차이, Tier와 evidence level이 별개의 축이라는 점을 인식해야 한다.
                      • Fusion 추출에서는 두 유전자의 breakpoint 좌표, supporting reads 수, 검출방법(DNA/RNA)이 핵심 변수다.
                      • 판단이 필요한 애매한 케이스(Tier 미표기, 모든 변이 혼합, 서술적 분류, 비표준 MSI 표기, 패널명 불명 등)에 대한 의사결정 규칙을 사전에 숙지하고, 규칙에 따라 일관되게 처리해야 통일된 데이터셋을 구축할 수 있다.
