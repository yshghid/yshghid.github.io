                  #Q1 Python을 사용할 때 가장 자주 사용하는 라이브러리는 무엇이고, 그 이유는 무엇인가요?

                  #A1

                  제가 Python을 사용할 때 가장 자주 사용하는 라이브러리는 Pandas와 pickle입니다.

                  Pandas의 장점은 데이터프레임을 기반으로 데이터를 확인 및 가공하는데 유용하다는 점입니다. 예를 들어 covid 19 프로젝트에서 1600개 사이토카인 발현량을 확인할 때 groupby 함수를 사용해서 환자의 메타데이터에 따라 데이터를 쉽게 확인할 수 있었습니다.

                  Pickle의 가장 큰 장점 중 하나는 대용량 Python 객체를 빠르게 저장하고 복원할 수 있다는 점입니다. 예를 들어 covid 19 프로젝트에서 전처리 완료된 대형 데이터셋이나 학습된 모델 객체를 .pkl로 저장해두면, 반복 실험 시 초기부터 다시 처리하지 않고 바로 이어서 작업할 수 있어 실험 효율이 크게 향상됩니다.

                  #Q2 Pandas와 NumPy의 차이점과 각각의 장점을 설명해주세요.

                  Pandas와 NumPy는 모두 데이터 분석에서 핵심적인 Python 라이브러리지만, 목적과 데이터 구조에서 명확한 차이가 있습니다.

                  NumPy는 ndarray라는 고속 수치 배열 객체를 기반으로 수학적 연산, 벡터화, 선형대수 처리에 특화된 라이브러리입니다. 특히 대규모 수치 데이터에 대해 연산을 빠르게 수행할 수 있다는 장점이 있습니다. 예를 들어, COVID-19 환자의 사이토카인 데이터를 정규화할 때 np.log1p()를 사용하여 로그 변환 기반 스케일링을 수행하였고, 이는 음수 값을 방지하면서 분포 왜곡을 줄이는 데 효과적이었습니다.

                  반면 Pandas는 DataFrame과 Series 객체를 기반으로 레이블이 지정된 테이블형 데이터를 직관적으로 다루는 데 유리합니다. COVID-19 프로젝트에서는 다양한 사이토카인 데이터를 환자별로 통합하여 관리하기 위해 Pandas의 groupby()와 merge() 함수를 적극 활용하였습니다. 이를 통해 열 이름 기반으로 통계값을 집계하거나, 다수의 환자 데이터를 병합하여 전처리하는 작업을 효과적으로 수행할 수 있었습니다.

                  요약하자면, NumPy는 수치 연산에 적합하고, Pandas는 테이블형 데이터를 다루는 데 강점이 있습니다. 두 라이브러리는 보완적으로 사용되며, 실제 프로젝트에서도 함께 활용되어 분석 효율을 높였습니다.
