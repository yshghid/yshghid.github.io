                분석 기획 방향성 도출 #

                • 분석 기획의 특징

                  1. 분석 기획: 실제 분석을 수행하기에 앞서 분석을 수행할 과제를 정의하고 의도했던 결과를 도출할수 있도록 이를 적절하게 관리하는 방안을 사전에 계획하는 작업
                  2. 데이터사이언티스트의 역량: 수학/통계학적 지식, 정보기술(IT기술, 해킹기술, 통신기술), 비즈니스에 대한 이해와 전문성

                • 분석 대상과 방법: 분석은 분석의 대상과 방법에 따라 4가지로 분류 가능.

                  1. 대상, 방법 모두 알때: Optimization
                  2. 대상, 방법 모두 모를때: Discovery
                  3. 대상만 알때: Solution
                  4. 방법만 알때: Insight

                • 목표 시점별 분석 기획 방안

                  1. 당면한 주제 해결: Speed&Test, Quick&Win, Problem solving
                  2. 지속적 분석, 내재화: Accuracy&Disploy, Long Term View, Problem Definition

                • 분석 기획시 고려사항

                  • 가용 데이터 고려
                  • 가치가 창출될수있는 적절한 활용방안, use case 탐색
                  • 장애요소에 대한 사전계획 수립

                분석 방법론 #

                • 분석 방법론 개요

                  1. 기업의 합리적 의사결정을 가로막는 장애요소
                    • 고정 관념, 편향된 생각, 프레이밍 효과
                  2. 적용 업무에 따른 몸델
                    • 폭포수 모델
                    • 프로토타입 모델
                    • 나선형 모델

                • KDD 분석 방법론

                  1. 데이터셋 선택
                  2. 데이터 전처리
                  3. 데이터 변환
                  4. 데이터 마이닝
                  5. 결과 평가

                • CRISP-DM 분석 방법론

                  1. 업무 이해
                  2. 데이터 이해
                  3. 데이터 준비
                  4. 모델링
                  5. 평가
                  6. 전개

                • 빅데이터 분석 방법론

                  • 빅데이터 분석의 계층적 프로세스
                  • 빅데이터 분석 방법론의 5단계
                    1. 분석 기획
                    2. 데이터 준비
                    3. 데이터 분석
                    4. 시스템 구현
                    5. 평가 및 전개

                분석 과제 발굴 #

                • 분석 과제 발굴 방법론

                  1. 하향식 접근방식: 분석 과제가 주어지고 단게화되어 수행.
                  2. 상향식 접근방식: 문제 정의가 어려운 경우, 데이터를 기반으로 문제를 개선하는 방식.

                • 하향식 접근방식: 문제 탐색 -> 문제 정의 -> 해결방안 탐색 -> 타당성 검토

                  1. 문제 탐색
                    • 비즈니스 모델 기반 문제 탐색: 업무, 제품, 고객, 규제와 감사, 자원 인프라 영역으로 기업의 비즈니스를 분석
                    • 외부 참조 모델 기반 문제 탐색: 유사, 동종 사례를 벤치마킹
                    • 분석 유즈 케이스 정의
                  2. 문제 정의: 비즈니스 문제를 데이터의 문제로 변환하여 정의
                  3. 해결방안 탐색: 분석역량(Who), 분석기법 및 시스템(How)으로 해결방안 탐색
                  4. 타당성 검토: 경제적 타당성, 데이터 및 기술적 타당성 검토

                • 상향식 접근방식

                  1. 정의
                    • 기업이 보유하고있는 다양한 원천데이터로부터 분석을 통해 통찰과 지식을 얻는 접근법
                    • 다양한 원천데이터를 대상으로 분석을 수행해서 가치있는 모든 문제를 도출하는 과정
                  2. 상향식 접근법의 특징
                    • 하향식 접근법은 논리적 단계별 접근법. 복잡하고 다양한 환경에서 발생하는 문제를 해결하기 어렵다.
                    • 디자인적 사고 접근법. WHY->WHAT 접근법. 데이터 자체를 객관적으로 관찰하여 문제를 해결.
                    • 상향식 접근법은 비지도 학습방법으로 수행. 데이터 자체의 결합, 연관성, 유사성을 중심으로 접근.
                    • 시행착올르 통한 문제해결: 프로토타이핑 접근법

                • 분석과제 정의

                  • 분석과제 정의서를 통해 분석별 필요 소스 데이터, 분석 방법, 데이터 입수 및 분석 난이도, 분석 수행주기, 검증 오너십, 상세분석과정 등을 정의

                분석 프로젝트 관리 방안 #

                • 분석과제 관리를 위한 5가지 주요 영역
                  • 분석프로젝트는 범위, 일정, 품질, 리스크, 의사소통 등 영역별 관리가 수행되어야함 / 데이터에 기반한 분석 기법을 적용함에 따라 분석과제관리 5가지 영역을 고려해서 관리가 필요.
                    데이터 복잡도, 속도, 사이즈, 분석 복잡도, 정확도
                • 분석프로젝트의 특성
                  • 분석가의 목표: 개별적인 분석업무 수행 + 전반적인 프로젝트 관리
                  • 분석가의 입장: 데이터 영역과 비즈니스 영역의 현황 이해, 프로젝트의 목표인 분석의 정확도 달성, 결과에 대한 가치 이해를 전달하는 조정자로서의 역할.
                  • 분석 프로젝트는 도출된 결과의 재해석을 통한 지속적인 반복및 정규화가 수행됨 -> 프로토타이핑 방식의 에자일 프로젝트 관리방식에 대한 고려도 필요.

                마스터 플랜 수립 프레임워크 #

                • 마스터 플랜 수립 프레임워크

                  • 분석 과제를 대상으로 ‘다양한 기준’을 고려해 적용 우선순위를 설정하고 분석 로드맵 수립
                  • 우선순위 고려요소: 전략적 중요도, 비즈니스 성과/ROI, 실행 용이성
                  • 적용범위/방식 고려요소: 업무 내재화 적용 수준, 분석 데이터 적용 수준, 기술 적용 수준

                • 우선순위 평가에 활용하기 위한 ROI 관점에서 빅데이터의 핵심 특징

                  1. 크기(Volume): 데이터의 규모/양
                  2. 다양성(Variaty): 데이터 종류/유형
                  3. 속도(Velocity): 데이터 생성속도/처리속도
                  4. 가치(Value): 분석 결과 활용 및 실행을 통한 비즈니스 가치
                  • 1, 2, 3 -> 투자비용 요소(Investment)
                  • 4 -> 비즈니스 효과(Return)
