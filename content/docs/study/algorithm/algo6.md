# #5 타겟 넘버

#2025-07-19

---


문제: n개의 음이 아닌 정수들이 있습니다. 이 정수들을 순서를 바꾸지 않고 적절히 더하거나 빼서 타겟 넘버를 만들려고 합니다. 사용할 수 있는 숫자가 담긴 배열 `numbers`, 타겟 넘버 `target`이 매개변수로 주어질 때 숫자를 적절히 더하고 빼서 타겟 넘버를 만드는 방법의 수를 return 하도록 solution 함수를 작성해주세요.

제한사항: 주어지는 숫자의 개수는 2개 이상 20개 이하입니다. 각 숫자는 1 이상 50 이하인 자연수입니다. 타겟 넘버는 1 이상 1000 이하인 자연수입니다.

입력: `numbers=[1,1,1,1,1]`, `target=3`

출력: `5`

풀이:

주어진 숫자 리스트 `numbers`를 순서를 바꾸지 않고 앞에서부터 차례대로 하나씩 더하거나 빼며, 최종적으로 목표 숫자인 `target`에 도달하는 경우의 수를 구한다. 문제에서 주어진 입력은 `numbers=[1,1,1,1,1]`, `target=3`이다. 모든 숫자가 1이므로 각각의 숫자에 대해 ‘+1’ 또는 ‘-1’로 분기하는 이진 트리 구조의 탐색이 이루어진다. 초기에는 인덱스 0에서 시작하며, 누적된 합은 0이다. 첫 번째 숫자 1에 대해 ‘+1’을 선택할 수도 있고, ‘-1’을 선택할 수도 있다. 따라서 이 시점에서 두 갈래의 경로가 만들어진다. 이후 두 번째 숫자에 대해서도 같은 방식으로 다시 두 갈래씩의 분기가 생기며, 모든 숫자를 다 사용했을 때 누적된 합이 `target` 값인 3이 되면 그것은 하나의 유효한 경로가 된다. 이 과정을 깊이 우선 탐색으로 구현하면, 각 단계에서 `dfs(index, total)`이라는 함수를 통해 현재 위치와 누적합을 추적하면서 재귀적으로 모든 가능한 경우를 탐색하게 된다.

이제 이 DFS 과정이 어떻게 전개되는지를 순서대로 자세히 살펴본다. 우선 `dfs(0, 0)`부터 시작한다. 인덱스가 0이고, 현재까지 합은 0이다. 이 상태에서 두 가지 선택을 한다. `+1`을 선택하면 `dfs(1, 1)`, `-1`을 선택하면 `dfs(1, -1)`로 이어진다. 먼저 `dfs(1, 1)`을 따라가 보면, 이 경로는 첫 번째 숫자를 +1로 선택한 경로이며, 두 번째 숫자에 대해 다시 두 가지 선택이 있다. `+1`을 선택하면 `dfs(2, 2)`, `-1`을 선택하면 `dfs(2, 0)`이 된다. `dfs(2, 2)`는 지금까지의 경로가 `+1, +1`인 경우이다. 세 번째 숫자도 +1로 선택하면 `dfs(3, 3)`, -1이면 `dfs(3, 1)`이 된다. 이 과정을 이어가면 다음과 같은 경로가 만들어진다: `dfs(3, 3)` → `+1` → `dfs(4, 4)` → `+1` → `dfs(5, 5)` → 끝, 합은 5 (불일치), `dfs(3, 3)` → `1` → `dfs(4, 2)` → `+1` → `dfs(5, 3)` → 끝, 합은 3 (정답 1개 확보). 이렇게 `dfs(2, 2)`로부터 시작된 분기에서는 `dfs(4, 2)` → `dfs(5, 3)`이 정답에 해당한다. 이제 다시 위로 돌아가서 `dfs(2, 0)`을 따라가 보면 `+1` → `dfs(3, 1)`, `-1` → `dfs(3, -1)`로 나뉜다. 이 경로는 처음 두 숫자를 `+1, -1`로 선택한 경우이다. 이어지는 분기 중 `dfs(3, 1)`에서 `+1` → `dfs(4, 2)` → `+1` → `dfs(5, 3)`로 또 정답 하나가 발견된다. 즉, `+1, -1, +1, +1, +1`과 같은 조합도 정답이다. 이런 식으로 `dfs(0, 0)`부터 시작해서 완전탐색을 수행하면 모든 경우의 수는 2⁵ = 32개이다. 이 중에서 누적합이 정확히 3인 경로를 센 것이 최종적인 답이 된다. 이 문제의 경우 유효한 경로는 다음 다섯 가지이다: `+1 +1 +1 -1 +1` → 합: 3, `+1 +1 -1 +1 +1` → 합: 3, `+1 -1 +1 +1 +1` → 합: 3, `1 +1 +1 +1 +1` → 합: 3, `+1 +1 +1 +1 -1` → 합: 3.

입력 2: `numbers = [4, 1, 2, 1]`, `target = 4`

출력 2: `5`

풀이 2:

입력으로 `numbers = [4, 1, 2, 1]`, `target = 4`가 주어졌으며, 우리는 이 네 개의 숫자에 대해 더하기 또는 빼기를 적용하여 결과적으로 4가 되는 경우의 수를 찾는다. 탐색은 `dfs(index, total)` 형태로 진행되며, `index`는 현재 몇 번째 숫자를 보고 있는지를, `total`은 현재까지 계산된 합을 의미한다. 초기 호출은 `dfs(0, 0)`이다. 이 의미는, 아직 어떤 숫자도 사용하지 않았고, 현재까지 합은 0이라는 뜻이다.

이제 이 DFS 탐색이 실제로 어떻게 전개되는지를 하나하나 따라가 보자. 가장 처음에는 `dfs(0, 0)`으로 시작한다. `index = 0`이고, `total = 0`이다. 이때 선택할 수 있는 건 `numbers[0] = 4`에 대해 더하거나 빼는 것이다. 따라서 다음 단계로는 `dfs(1, 4)` 혹은 `dfs(1, -4)`가 된다. 첫 번째 숫자를 +4 혹은 -4로 썼다는 의미다. 먼저 `dfs(1, 4)`로 들어가 보자. 여기서 두 번째 숫자는 1이므로, 다시 +1 또는 -1을 적용할 수 있다. 그러면 `dfs(2, 5)` 또는 `dfs(2, 3)`이 된다. 예를 들어 `dfs(2, 5)`는 지금까지 4 + 1 = 5라는 누적합을 의미하고, `dfs(2, 3)`는 4 - 1 = 3이라는 누적합을 의미한다. 다음으로 `dfs(2, 5)`를 보자. 세 번째 숫자인 2에 대해 +2 또는 -2를 선택하면, `dfs(3, 7)` 또는 `dfs(3, 3)`이 된다. 이어서 네 번째 숫자인 1에 대해 +1, -1을 적용해 `dfs(4, 8)` 또는 `dfs(4, 6)` (전자가 `dfs(3, 7)`에서 오는 경우), 또는 `dfs(4, 4)` 또는 `dfs(4, 2)` (`dfs(3, 3)`에서 오는 경우)가 된다. 이 중에서 `dfs(4, 4)`는 `index == len(numbers)`이므로 탐색을 종료하고, `total == target`인지 확인한다. 현재 `total = 4`, `target = 4`이므로 정답 하나를 찾은 것이다. 이 경로는 다음과 같다: +4 → +1 → -2 → +1, 즉 `4 + 1 - 2 + 1 = 4`. 다시 돌아가서 `dfs(2, 3)`을 보면, 이 경로는 4 + (-1) = 3까지 온 것이다. 여기에 세 번째 숫자 2를 더하거나 빼면 `dfs(3, 5)` 또는 `dfs(3, 1)`이 되고, 이어서 네 번째 숫자 1을 더하거나 빼면 각각 `dfs(4, 6)`, `dfs(4, 4)` 또는 `dfs(4, 2)`, `dfs(4, 0)`이 된다. 이 중에서도 `dfs(4, 4)`가 정답이다. 이 경로는 +4 → -1 → +2 → -1, 즉 `4 - 1 + 2 - 1 = 4`로 두 번째 정답 경로다. 이제 위로 올라가서 `dfs(1, -4)`의 경우도 살펴본다. 이 경로는 첫 번째 숫자를 음수로 선택한 경로로, 현재까지 합은 -4다. 이어서 +1 또는 -1을 하면 `dfs(2, -3)` 또는 `dfs(2, -5)`가 된다. 이 아래로도 계속 탐색을 하게 되지만, 이 분기들에서는 끝까지 가도 최종 합이 4가 되지 않는다. 실제로 `-4 ± 1 ± 2 ± 1`을 해도 4는 만들 수 없다. 이 과정을 나무(Tree) 구조로 표현하면 DFS 트리가 총 2^4 = 16개의 말단 노드를 가지며, 각각의 경로에서 `index == len(numbers)`에 도달했을 때 합이 4인 경우만 카운트된다. 실제로 모든 탐색 경로 중에서 정확히 합이 4가 되는 경로는 +4 +1 -2 +1 = 4, +4 -1 +2 -1 = 4 총 두 가지이다. 따라서 최종 출력 값은 2가 된다.

정답:

```python
def solution(numbers, target):
    answer = 0

    def dfs(index, total):
        nonlocal answer
        if index == len(numbers):
            if total == target:
                answer += 1
            return
        dfs(index + 1, total + numbers[index])  # 현재 숫자를 더함
        dfs(index + 1, total - numbers[index])  # 현재 숫자를 뺌

    dfs(0, 0)
    return answer

# 테스트
print(solution([1, 1, 1, 1, 1], 3))  # 출력: 5
```
